## Context — why this exists

Wave 1 shipped our first market-neutral, multi-asset strategy (Gatev et al. 2006
distance pairs) and, to do it, taught the backtest engine to run a strategy across
**two** synchronized data feeds (`engine.run_pairs_backtest`). That capability
unblocks an entire family of strategies we previously couldn't express. It also
exposed our biggest remaining gap: we **cannot do cross-sectional strategies**
(rank a universe, long winners / short losers) — which is why Jegadeesh & Titman
had to be dropped from wave 1 rather than mis-attributed.

This roadmap closes that gap and grows the multi-asset shelf, keeping the same
rigor-as-wedge discipline: every strategy is paper-grounded, run on real data, and
gated by DSR/PBO/OOS before it can claim it works.

## The architectural fork: how many feeds?

The current `run_pairs_backtest` handles **exactly two** bar-aligned feeds. The wave
splits on this:

- **Phase 1** strategies run on the existing 2-feed engine — no engine change.
- **Phase 2 & 3** need an **N-feed engine** (rank/weight across a universe).

### Engine milestone (prerequisite for Phase 2+)

Generalize `run_pairs_backtest` → **`run_multi_backtest(prices_list, *, strategy_cls, ...)`**:
- Inner-join N price frames on their common date index; add N `PandasData` feeds.
- Reuse the existing `_add_analyzers` / `_extract_result` path so metrics stay
  identical across single / pair / N-asset runs.
- Strategies access `self.datas[i]`, compute cross-sectional ranks/weights each bar,
  and `order_target_size(data=..., target=...)` per leg.

**Acceptance:** a hermetic `test_multi_engine.py` proving an N-feed strategy runs,
aligns on common dates, and errors on disjoint dates (mirror `test_pairs_engine.py`).
Estimated ~1 day. Ship as its own PR (review-isolated — it touches the shared runner).

## Phase 1 — Pairs upgrades (NO engine change)

Cheapest wins; the 2-feed engine is ready today. Target: +3–4 strategies.

| # | Strategy | Paper anchor (verify numbers at build time) | What it adds over Gatev distance |
|---|---|---|---|
| 1.1 | **Cointegration pairs** (Engle-Granger test + Ornstein-Uhlenbeck half-life) | Engle & Granger (1987), *Econometrica* 55(2):251-276, DOI 10.2307/1913236; Vidyamurthy (2004), *Pairs Trading: Quantitative Methods and Analysis* (book) | Replaces the distance/z-score heuristic with a statistically *tested* long-run relationship and a principled mean-reversion speed (OU half-life). The "proper" stat-arb method. |
| 1.2 | **Kalman-filter dynamic hedge ratio** | Elliott, van der Hoek & Malcolm (2005), *Quantitative Finance* 5(3):271-276, DOI 10.1080/14697680500149370 | Time-varying hedge ratio instead of a fixed one — handles drifting relationships, the #1 cause of static-pair blow-ups. |
| 1.3 | **Additional economic pairs** (data-only; reuse `PairsDistanceTrading`) | — (same Gatev anchor) | KO/PEP (same-sector consumer staples), EWA/EWC (the textbook cointegration pair — Australia/Canada), GLD/SLV (gold/silver ratio). New tickers + fixture entries only. |

**Per-strategy acceptance (all phases):**
- New `analytics-engine/strategies/<name>.py` with full passport metadata (template:
  `gatev_2006_pairs_distance.py`), including `REGIME_TAG`.
- Real-data fixture entry generated by `scripts/regen_fixtures.py` (add-only; never
  hand-typed; never overwrite legacy entries).
- `paper_claimed_*` only when the source reports a clean, *verified* number; else null.
- Hermetic tests; `uv run pytest` green; backend `test_strategy_endpoints.py` extended.
- DSR/PBO/OOS computed; honest CANDIDATE status until it actually clears the gate.

## Phase 2 — Cross-sectional & portfolio (REQUIRES N-feed engine)

The high-value tier. This is where **Jegadeesh & Titman is finally done correctly**.

| # | Strategy | Paper anchor (verify numbers at build time) | Engine need |
|---|---|---|---|
| 2.1 | **Cross-sectional momentum** | Jegadeesh & Titman (1993), *J. Finance* 48(1):65-91, DOI 10.1111/j.1540-6261.1993.tb04702.x | Rank universe by trailing return; long winners / short losers; periodic rebalance |
| 2.2 | **Dual momentum** | Antonacci (2014), *Dual Momentum Investing* (book; SSRN working paper 2042750) | Relative momentum across assets + absolute-momentum cash filter |

## Sequencing

1. **Phase 1** first — zero engine risk, immediate library growth.
2. **Engine milestone** as its own PR.
3. **Phase 2** — retires the "no cross-sectional" gap (highest strategic value).
4. **Phase 3** if time allows.

## Non-negotiable discipline (carried from wave 1)

- **Verify every paper's claimed numbers before populating `paper_claimed_*`.** Never
  guess. If the source reports no clean Sharpe (common for books), leave it null.
- **Provenance honesty over coverage.** If a strategy can't be implemented faithfully
  to its cited paper (e.g. it's cross-sectional but the engine is single-asset), drop
  it rather than mis-attribute. Wave 1 dropped Jegadeesh 1990 and Lo-Mamaysky-Wang 2000
  for exactly this reason.
- **All fixture metrics computed from real backtests** via `scripts/regen_fixtures.py`.
  Add-only — never overwrite legacy entries (they don't reproduce on current data).
- **Honest gate verdicts.** A strategy that fails DSR/PBO/OOS stays CANDIDATE and says
  so. That honesty *is* the product.
